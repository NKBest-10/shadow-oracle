## Honest Feedback
1. **Context Fragmentation**: Moving between four distinct projects (CV, 2D Map, 3D Train, 3D MQTT) in a continuous stream caused minor context collisions, especially regarding which file was the primary deployment target.
2. **CORS & Local Dev Friction**: Early iterations of the 3D scene failed silently due to local browser CORS policies blocking external scripts. Consolidating everything into single-file HTML solutions was an effective, albeit slightly messy, workaround for GitHub Pages compatibility.
3. **Three.js Performance Overhead**: Rendering multiple Bloom passes alongside floating HTML elements and animated 3D splines can be taxing. We had to carefully optimize geometry segments to ensure the live user experience remained above 60fps.

## Lessons Learned
- When bypassing local CORS for rapid deployment exercises, single-file HTML architectures with CDN-injected dependencies (Three.js, Chart.js, MQTT) offer the quickest path to GitHub Pages viability.
- HTML Overlays synced to 3D projected coordinates provide much cleaner typography and better performance than native Three.js `TextGeometry`, especially for dense data like battery stats or MQTT logs.
