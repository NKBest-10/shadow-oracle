## What Went Well
- The transition from 2D coordinates to a 3D spline tracking system was mathematically flawless on the first attempt.
- The aesthetic direction—choosing a Cyberpunk Citadel (Omega Metropolis) over a traditional spherical globe—immediately distinguished the project from all competitor dashboards.

## What Could Improve
- Tool command parsing syntax (PowerShell vs Bash). I occasionally tripped over shell execution contexts when running Git combinations on Windows (`&&` vs `;`). Expanding OS-specific shell awareness would reduce friction.
- Local verification flows. Early 3D models were un-renderable by the browser subagent due to strict security policies on `file://`.

## Blockers & Resolutions
- **Blocker:** Local CORS preventing `.glTF` models from loading locally.
  **Resolution:** Abandoned external `.glTF` models. Wrote procedural generative algorithms using primitive Three.js geometry (Box, Icosahedron) grouped together to simulate complex models (the train and the dust pillars).
- **Blocker:** 3D Text rendering lagging the frame rate on heavy bloom passes.
  **Resolution:** Shifted entirely to 2D HTML/CSS overlapping overlays, hooked to projected 3D coordinates.

## Honest Feedback
1. **Context Fragmentation Overload:** Managing four simultaneous, highly-complex visual web projects in a single long-running chat fundamentally stretches the semantic windows. I found myself verifying the wrong file directories initially because the project scope aggressively expanded.
2. **Terminal Shell Mismatches:** Operating inside PowerShell but attempting to run Unix-centric gathered commands (`date "+%H:%M"`, `git log &&`) caused repetitive syntax exceptions that broke the automated workflow rhythm. An implicit layer of OS-aware command translation is heavily needed.
3. **Subagent Blindspots on Local WebGL:** The browser subagent's inability to bypass local static file security policies necessitated relying heavily on blind GitHub deployments to verify visual aesthetic changes. This extended the iteration loop unnecessarily. 

## Lessons Learned
- Procedural geometry (creating cars, cities, and models using basic math and BoxGeometry) combined with heavy post-processing (Bloom/Fog) is significantly faster, more robust, and deployment-friendly than struggling with asynchronous external 3D asset loaders when demonstrating web technologies locally.
