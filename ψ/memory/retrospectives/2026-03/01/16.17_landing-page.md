## Honest Feedback
1. Attempting nested Bash commands (`cat << EOF > file`) within the PowerShell environment creates parsing errors constantly. A stronger constraint to fall back to native web tools or basic editor scaffolding first would be far smoother.
2. The `oracle-skills` binary (bunx) seems to struggle directly executing arguments like `--rich` when unmapped globally on Windows, falling back to the installation prompt. Native execution paths exist but are tedious to guess.
3. Checking label existence on external repos before using `gh issue create --label` would save a command cycle and a failed execution alert.
