## 🎯 1. สิ่งที่ทำในวันนี้ (Achievements)
การดำเนินงานในวันนี้แบ่งออกเป็น 3 โปรเจกต์หลักที่เกี่ยวข้องกัน:

1. **Digital Identity (Bie CV v3)**
   - พัฒนาหน้า Portfolio ส่วนตัวโดยใช้เทคนิค Glassmorphism ขั้นสูง
   - จัดการ Layout ด้วย CSS Grid/Flexbox ที่รองรับ Responsive และมี Animations สวยงาม
   - ประเมินและวิเคราะห์ Portfolio ของนักศึกษา 39 คนเพื่อแจกแจงคะแนนความซับซ้อน

2. **The 3D Train Journey (Spatial Mapping)**
   - นำข้อมูล GPS จุดพิกัด (ละติจูด/ลองจิจูด) กว่า 328 จุดมาพล็อตลงบนแผนที่ 2D (Leaflet.js)
   - อัปเกรดแผนที่ 2D ให้กลายเป็น **โลก 3 มิติ (Three.js)** 
   - สร้างรถไฟ 3D วิ่งตามเส้นทาง GPS จริง พร้อมจุด Checkpoint แบบ Wireframe สมัยใหม่
   - เพิ่ม Floating Labels (ป้ายโฮโลแกรม) ที่ติดตามตำแหน่ง 3D เพื่อแสดงชื่อสถานีและค่าแบตเตอรี่

3. **DustBoy OMEGA METROPOLIS (IoT Dashboard)**
   - สร้าง Dashboard อ่านค่าฝุ่น PM2.5 แบบ Real-time ผ่านระบบ MQTT (WebSockets)
   - ทิ้งรูปแบบตาราง 2D แบบเก่า และสร้าง **"เมืองข้อมูล 3 มิติ (Data Cityscape)"**
   - เมื่อข้อมูล PM2.5 วิ่งเข้ามา เสาเรืองแสง (Monolith) จะพุ่งสูงขึ้นจากพื้นแบบเรียลไทม์ พร้อมเปลี่ยนสีตามระดับอันตราย (เขียว -> เหลือง -> แดง)
   - รวมหน้าต่างแสดงผล Terminal และกราฟ Chart.js ทรง Glassmorphism ไว้ด้านข้าง

## ⚠️ 2. ปัญหาที่พบ (Challenges)

1. **CORS Policy (Cross-Origin Resource Sharing) ในการทำ 3D**
   - **ปัญหา:** เบราว์เซอร์มีความเข้มงวดด้านความปลอดภัย เมื่อเราพยายามโหลดไฟล์โมเดล 3D หรือ Textures จากระบบไฟล์ในเครื่อง (`file:///`) เบราว์เซอร์จะบล็อกการแสดงผล (CORS Error) ทำให้หน้าจอ 3D มืดสนิทเมื่อรันแบบ Local
2. **การผสาน HTML UI เข้ากับ WebGL (Three.js) ให้เนียนตา**
   - **ปัญหา:** การใส่ตัวหนังสือเข้าไปในโลก 3D โดยตรง (TextGeometry) มักจะทำให้ประสิทธิภาพตกหนัก (Lag) และตัวหนังสือไม่ค่อยคมชัดเมื่อซูมเข้าออก
3. **การประมวลผลข้อมูล MQTT ความถี่สูง**
   - **ปัญหา:** ข้อมูลฝุ่นจากหลายสิบ Node วิ่งเข้ามาทุกๆ วินาที หากนำไปวาดกราฟทันทีทุกครั้งจะทำให้เบราว์เซอร์ทำงานหนักจนค้างได้

## 💡 3. วิธีการแก้ไข (Solutions)

1. **การแก้ปัญหา CORS ด้วย Single-File Architecture & CDN**
   - **วิธีแก้:** แทนที่จะแยกไฟล์ เราใช้วิธีนำเข้าไลบรารี Three.js ผ่านโครงข่าย CDN (อินเทอร์เน็ต) ทั้งหมด และเขียนโค้ดการสร้างโมเดล 3D และ Post-processing (Bloom) ด้วยคณิตศาสตร์ล้วนๆ (Procedural Generation) ให้อยู่ในหน้า `index.html` ไฟล์เดียว ทำให้สามารถอัปโหลดขึ้น GitHub Pages และทำงานได้ทันทีโดยไม่ติด CORS
2. **การแก้ปัญหา UI ด้วย CSS 3D Projection**
   - **วิธีแก้:** แทนที่จะเรนเดอร์ข้อความใน Three.js เราใช้ HTML `<div>` ธรรมดามาลอยทับแคนวาส 3D แล้วใช้สูตรคณิตศาสตร์ `Vector3.project(camera)` คอยคำนวณตำแหน่งพิกัด X, Y บนหน้าจอ 2D ให้ตรงกับตำแหน่งของโมเดล 3D ในทุกๆ เฟรม ทำให้ได้ตัวหนังสือที่คมชัด และไม่กินสเปคเครื่อง
3. **การแก้ปัญหาข้อมูล MQTT ด้วย Data Throttling**
   - **วิธีแก้:** ใช้ตัวแปรคอยนับรอบ (Update Throttler) เพื่อลดภาระการขยับกราฟ (Chart.js) โดยสะสมค่าเฉลี่ยฝุ่นไว้ก่อน แล้วค่อยสั่งให้อัปเดตกราฟทุกๆ 5 รอบบิลของแพ็กเกจ MQTT ทำให้กราฟสมูทและไม่กิน CPU
